The Residual Stream: A Unified View of How Transformers Think

This blog is inspired by Distill-style writing. I believe it is valuable to revisit foundational ideas carefully and from a fresh perspective, and that is the main goal of this post. I hope readers will find this viewpoint both intuitive and useful.

Modern transformer models can look complicated: stacked attention heads, feed-forward networks, layer norms, skip connections. But beneath that engineering complexity lies a surprisingly elegant story about what these components are doing. At a high level, they are all participating in the gradual refinement of a shared, evolving representation, one additive update at a time.

This post develops that picture carefully. I will argue that both attention and MLP layers can be viewed as mechanisms for reading from a shared scratchpad and writing targeted updates back into it. They differ not in whether they refine the representation, but in how they decide what to read and what to write.

The residual stream as a shared scratchpad

At the heart of every transformer is the residual stream: a vector at each token position that persists across all layers of the network. A useful mental picture is to think of it as a shared whiteboard or scratchpad that every layer can read from and write to.

The key point is that attention and MLP layers do not replace the residual stream. They only add to it. Formally, the update takes the form

\[ x_{\text{new}} = x_{\text{old}} + \mathrm{Layer}(x_{\text{old}}) \]

This is the role of the skip connection. If a layer has little to contribute, it can output a near-zero vector and leave the representation almost unchanged. Information written earlier therefore remains available to downstream computations. From this perspective, the forward pass of a transformer is a sequence of incremental edits to a persistent shared state.

A useful metaphor. The model is repeatedly revising the same draft rather than rewriting it from scratch. Each layer adds, sharpens, redirects, or partially cancels what is already present.

Attention: exchanging information globally, through weighted transport

Attention is the mechanism that allows token positions to exchange information. To make this concrete, it is worth tracing the computation for a single token at a single attention head.

  1. Query. The token forms a query vector \( q = W_Q x \), representing what information it is currently looking for.
  2. Keys. Every token forms a key vector \( k_j = W_K x_j \), representing what information it can offer.
  3. Scores. The relevance between the querying token and token \(j\) is measured by the dot product \( q \cdot k_j \). After scaling and softmax, these become attention weights \( \alpha_j \), which sum to \(1\).
  4. Values. Each token also produces a value vector \( v_j = W_V x_j \). The head retrieves the weighted sum \( \sum_j \alpha_j v_j \).
  5. Write-back. This result is projected by \( W_O \) and added back into the querying token’s residual stream.

Aside from the softmax gating, the actual information transfer is linear: the head writes back a weighted linear combination of value vectors. In this sense, attention acts like a routing rule for deciding which other tokens get to modify the current token’s state.

This is why attention is such a powerful mechanism for global information exchange. A token can directly incorporate information from faraway positions in the sequence, and it does so by adding a context-dependent update to its residual stream.

What about erasing? Attention cannot literally delete information. It can only add vectors. What looks like erasure is usually implemented by later layers adding vectors that partially cancel earlier ones.

The broader point is that the architecture strongly favors linear transport and linear composition, even though the gating itself is nonlinear. That makes linear structure in representation space especially useful for the model.

MLP layers: refinement through learned memory

If attention is the mechanism for exchanging information across positions, the MLP is the mechanism for refining each token’s representation using learned internal memory. For a single token, the MLP computes

\[ \mathrm{MLP}(x) = W_{\text{out}} \, f(W_{\text{in}}x) \]

This can be read in three steps.

  1. Pattern matching. The matrix \( W_{\text{in}} \) computes scores for many learned patterns by taking dot products with the current token representation.
  2. Gating. The nonlinearity \( f \) selects which of those patterns are active in the current context.
  3. Write-back. The matrix \( W_{\text{out}} \) turns the activated pattern scores into a weighted combination of output directions, which is then added back to the residual stream.

This is why MLPs are often interpreted as a kind of key-value memory. The rows of \( W_{\text{in}} \) act like keys, the output directions of \( W_{\text{out}} \) act like values, and the nonlinearity serves as a retrieval gate.

The important unifying point is that the MLP is also performing refinement by additive update. Unlike attention, it does not communicate across token positions. But it still reads the current residual state, computes what should be added, and writes that update back into the same shared stream.

MLP as knowledge storage

This key-value perspective suggests that MLP weights are not merely generic machinery. They can be understood as a form of learned knowledge storage. When the current token state matches the right pattern, the corresponding value directions are written into the stream.

MLP as a refinement filter

Because the MLP acts after attention within a block, it processes a representation that has already been enriched by contextual information from other tokens. In that sense, the MLP can be viewed as a refinement filter: it can amplify useful directions, add implications suggested by the new context, and sometimes help suppress less relevant information through later cancellation.

A compact summary. Attention assembles; MLP distills. Attention gathers context from elsewhere in the sequence, while the MLP transforms that enriched local state using learned internal structure.

A walkthrough: from zero to two layers

This section is inspired in part by A Mathematical Framework for Transformer Circuits, especially the idea that depth can be understood in terms of progressively richer in-context computation. My discussion here extends that picture by emphasizing the role of both attention and MLP updates in refining the residual stream.

The residual-stream view becomes especially clear when we trace how capability grows with depth.

Model What it can do How the residual stream is used
0-layer Token-to-output mapping only The embedding is read almost directly by the unembedding, \( x_{\text{out}} = W_U W_E x \). There is essentially no in-context processing.
1-layer One round of contextual modification A single attention layer can read earlier tokens and write context-dependent information into the stream. The MLP can then further refine that representation.
2-layer Composed in-context reasoning The second layer receives a stream already edited by the first. This allows more compositional behavior, because the first layer can shape the representation used by the second layer to form its queries, keys, and values.

One clean way to write a transformer block is

\[ a^{(\ell)} = x^{(\ell-1)} + \mathrm{Attn}_{\ell}(x^{(\ell-1)}) \] \[ x^{(\ell)} = a^{(\ell)} + \mathrm{MLP}_{\ell}(a^{(\ell)}) \]

Each additional layer adds another round of “read the current stream, compute an update, add it back.” The final representation is the accumulated result of these edits. This is gradual refinement made explicit in the algebra.

As depth increases, we should expect models to develop richer in-context algorithms. For example, more structured behaviors such as induction-style circuits only become available once enough layers exist to support this kind of composition.

Layer norm and the geometry of search

Without normalization, the residual stream could become unstable in scale as updates accumulate. If the scale of the representation drifts too much, then the resulting query, key, and value vectors may become poorly behaved relative to one another. Layer normalization helps keep the representation in a controlled regime before each attention or MLP block reads it.

Combined with residual connections, this improves optimization stability and gives gradients a more reliable path backward. Intuitively, normalization restricts the range of scales the model is allowed to operate in. One can think of this as making the model’s search through representation space more controlled and less chaotic.

I would treat this as a useful intuition rather than a fully precise theorem: normalization can be seen as stabilizing the geometry of the space in which refinement occurs. This stability is helpful for training, even if it sometimes makes mechanistic analysis harder because the scaling becomes data-dependent.

The above analysis suggests an interesting connection to recent architectural ideas that explicitly constrain how representations evolve. For example, the mHC paper can be read as exploring the broader design principle that controlling the geometry of updates may improve training stability. I do not mean that layer normalization and mHC are identical mechanisms. Rather, they seem to point in a similar conceptual direction: unrestricted representational growth can be unstable, and carefully constraining how the residual stream evolves may be a useful inductive bias.

A tradeoff. Normalization helps keep training stable, but it also makes the representation geometry harder to read off directly, since scale now depends on the current input.

The stickiness problem

The residual stream’s greatest strength is also a potential weakness. Information written early is not truly deleted. It remains present unless later layers spend capacity adding vectors that counteract it. This makes information “sticky”: easy to preserve, but sometimes costly to suppress.

This perspective suggests one possible explanation for why some layers may appear to perform similar or incremental computations. If the model operates on a shared persistent stream, later layers may repeatedly refine partially processed information rather than replacing it outright. That does not prove that all apparent redundancy comes from the residual stream, but it does make such behavior easier to understand.

This also raises an interesting speculative question: what would happen if different layers operated in more strongly differentiated representation spaces, rather than repeatedly editing a single shared one? My guess is that such a design could increase capacity in some settings, though this is only an intuition for now.

More broadly, one can ask whether future architectures should give themselves greater ability to reorganize, compress, or selectively preserve information as depth increases. Thinking in terms of residual-stream refinement may help make such design questions more concrete.

Linear representations: the expected geometry of a mostly linear machine

There is a natural reason to expect transformer representations to exhibit substantial linear structure. Both attention and MLP layers rely heavily on linear maps, dot products, and additive composition. If much of the computation proceeds through operations of this kind, it is unsurprising that useful features are often stored as directions in representation space that can be read out and combined linearly.

\[ \mathrm{vec}(\text{king}) - \mathrm{vec}(\text{man}) + \mathrm{vec}(\text{woman}) \approx \mathrm{vec}(\text{queen}) \]

Examples like this are not merely curiosities. They suggest that the architecture finds linear structure convenient and computationally accessible.

Nonlinear representations: the frontier

Of course, linearity cannot be the whole story. Deep models compose many stages of gated, context-dependent computation, and the resulting geometry can be far richer than a single linear map would suggest. One layer may write a rough approximation, later layers may reshape it, and the final representation can reflect a highly nonlinear computation built out of many locally simple steps.

In that sense, deep transformers can construct curved reasoning from repeated additive refinements.

All flows through the residual stream

Every piece of information a transformer processes, whether read from tokens, retrieved from learned memory, exchanged through attention, or composed across layers, ultimately lives in the residual stream. The stream is the model’s memory, workspace, and communication bus all at once.

input embeddings (layer 0) │ ├─ attn₁ writes context-dependent information across tokens ├─ mlp₁ refines each token using learned memory │ ├─ attn₂ reads a stream already shaped by layer 1 ├─ mlp₂ further distills and redirects the representation │ ├─ ... │ └─ unembedding reads the fully refined vector → next-token distribution

The overall process is one of progressive refinement: a rough input representation becomes a detailed, context-sensitive, knowledge-enriched vector at the output. Layer by layer, the model keeps editing the same shared stream.

This perspective is useful not only as an explanation of transformers, but also as a guide for interpretability. If so much of the model’s computation consists of writing into and reading from the residual stream, then understanding the dynamics of that stream may be more illuminating than treating attention heads and MLPs as isolated modules.
